# Spectra Rewards Distribution

This directory contains the Spectra report and repartition steps. The active sdSpectra merkle is generated by `script/sdTkns/generateUniversalMerkleSpectra.ts` and published as a Base sdToken merkle.

## Process

1. Claims are fetched from the Spectra Safe Module on Base by the claims pipeline.
2. `1_report.ts` generates `bounties-reports/{timestamp}/spectra.csv`.
3. `2_repartition.ts` calculates voter and delegator distribution data from the `sdspectra.eth` Snapshot proposal.
   - Output: `bounties-reports/{timestamp}/spectra/repartition.json`.
4. `../sdTkns/generateUniversalMerkleSpectra.ts` combines the current repartition with previous unclaimed balances.
   - Primary output: `bounties-reports/{timestamp}/sdTkns/sdtkns_merkle_8453.json`.
   - Compatibility output: `bounties-reports/{timestamp}/spectra/merkle_data.json`.
5. The `sdTokens: Merkle` workflow publishes the Base merkle to `bounties-reports/latest/sdTkns/sdtkns_merkle_8453.json`.

## Commands

```bash
pnpm spectra-report
pnpm spectra-repartition
pnpm sd-merkle:spectra

# Specific period
pnpm tsx script/sdTkns/generateUniversalMerkleSpectra.ts --timestamp 1771459200
```

## Notes

- There is no standalone `3_merkles.ts` in `script/spectra/`.
- Historical `spectra/merkle_data.json` files are kept for compatibility; current publish targets live under `sdTkns/`.
- Current publish targets live under `sdTkns/`; historical Spectra latest copies are no longer produced.
